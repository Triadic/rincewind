LoggerFactory

Description
Description | Vars | Methods (details)

Implements interfaces:

  • LoggerFactoryInterface

The logger factory exists, so all information needed for a logger is stored in here, and the factory can be passed to other objects (preferably via setLoggerFactory()).

Each object can then decide how many logger to use, and set the resource name individually, without having to care about the location the file is going to be saved to.

Most of the methods are the same as for the Logger, except of course log() and debug(). In general you set up the loggerFactory as you would a logger, and then get the logger out of it via getLogger()
