FileRetriever

Description
Description | Descendents | Methods (details)

The FileRetriever actually fetches a files, and returns the File object.

It's basically a list of static functions, which the File class extends, so normally you don't call them on the FileRetriever but rather the File class itself. The FileRetriever is a class on it's own, so it can be passed to other objects, so they don't have to call the static File:: functions (which makes them rather difficult to test.)
